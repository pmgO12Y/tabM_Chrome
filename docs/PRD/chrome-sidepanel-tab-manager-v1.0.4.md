# Chrome 侧边栏标签管理插件 PRD v1.0.4

## 1. Executive Summary
开发一款运行在 Chrome 侧边栏中的标签管理插件，面向 300+ 标签页的重度用户场景，提供按窗口浏览、快速搜索、切换、关闭和固定等基础能力。首版以高性能、低内存、模块化扩展为核心，优先解决“浏览器标签过多导致定位和切换效率下降”的问题，并为后续分组、批量操作和会话管理等功能预留清晰架构边界。

## 2. Problem Statement
- 多窗口、多任务并行的 Chrome 重度用户，需要在大量标签中快速定位目标页面。
- 原生横向标签栏难以同时表达窗口上下文和标签层级，切换成本高。
- 若扩展本身存在加载等待、视觉噪声或卡顿，会削弱标签管理工具的价值。

## 3. Target Users
- 主用户：100-300+ 标签的重度用户。
- 次用户：希望通过侧边栏获得更稳定浏览体验的普通用户。
- 典型任务：按窗口浏览、搜索定位、快速切换、关闭、固定标签。

## 4. Strategic Context
- Chrome Side Panel 已提供稳定承载容器。
- 首版聚焦高频操作和真实体感性能，而不是复杂整理能力。
- 视觉设计应优先服务信息密度、扫描效率和拉伸流畅度。

## 5. Solution Overview
### 产品边界
- 平台：Chrome Desktop。
- 技术基线：Manifest V3 + Side Panel API。
- 不兼容 Firefox / Edge，不提供 popup 或 new tab 版本。

### MVP 能力
- 按窗口展示标签列表，支持展开/折叠。
- 搜索标题与 URL。
- 切换标签、关闭标签、固定/取消固定。
- 查看当前窗口或全部窗口。
- 首屏未就绪时展示加载动画和标签加载进度。
- 浏览器当前激活标签变化时，侧边栏自动滚动定位到对应项。

### 视觉与交互约束
- 侧边栏 UI 采用无边界风格，避免卡片堆叠感。
- 普通列表项默认透明底色。
- 鼠标悬浮态使用浅灰底色。
- 选中态使用浅黄底色。
- 启动 loading 必须在 HTML 壳层即可见，不能依赖 React 首次挂载完成后才出现。

### 明确不做
- 会话保存与恢复。
- 批量操作。
- 自动整理规则。
- 云同步。
- 深度标签组管理。

## 6. Success Metrics
- 300+ 标签场景下首次可交互渲染不高于 500ms。
- 1000 标签场景下若首屏尚未可见，必须显示加载状态与进度。
- 搜索输入到结果刷新不高于 100ms。
- 单个标签事件到 UI 反映不高于 50ms。
- 侧边栏宽度拖拽时不得因视觉特效产生明显卡顿。

## 7. User Stories / Acceptance Criteria
- 作为用户，我可以按窗口查看标签，以理解当前工作上下文。
- 作为用户，我可以搜索标题或 URL，以更快定位目标标签。
- 作为用户，我可以直接切换、关闭或固定一个标签，而不必回到顶部标签栏。
- 作为用户，我可以只看当前窗口，也可以切换到全部窗口视图。
- 作为用户，当浏览器当前激活标签变化时，侧边栏会自动滚动到对应标签位置。
- 作为用户，当首屏尚未准备完成时，我能看到明确的加载动画和 `已加载 / 总数` 进度。
- 验收标准：
  - 搜索命中结果不得被折叠态隐藏。
  - 当前激活标签必须有清晰高亮。
  - 启动空白期必须由 HTML 壳层 loading 覆盖。
  - 普通、悬浮、选中三态颜色必须稳定可辨识。

## 8. Out of Scope
- 保存和恢复整组标签页。
- 多选和批量整理。
- 智能分类、规则引擎或 AI 功能。
- 账号同步与跨设备同步。

## 9. Risks / Dependencies
- 依赖 Chrome Side Panel API、Manifest V3 service worker、`chrome.tabs` 和 `chrome.storage`。
- MV3 service worker 可能被挂起，需要保证状态快速重建。
- 大量标签与宽度拖拽会放大重排和重绘成本。
- 无边界风格若层级处理不清，会造成信息辨识度下降。

## 10. Open Questions
- 后续是否需要在稳定基础上重新引入更可靠的列表虚拟化。
- 是否需要提供更强的紧凑模式或密度切换。
- 后续批量操作应以多选模式还是命令面板模式承载。

## 11. Verification Plan
- 自动验证：
  - 单元测试覆盖状态归并、搜索展示规则、偏好增量更新、favicon 安全回退。
  - 类型检查确保消息协议、偏好类型和状态结构一致。
  - 构建产物必须可生成 `background` 与 `sidepanel` 两个入口。
- 手工验证：
  - 在 100+、300+、1000 标签场景下验证首次渲染、搜索和单 tab 事件响应。
  - 在启动瞬间验证 HTML 壳层 loading 先于 React 内容出现。
  - 验证 hover 为浅灰、active 为浅黄。
  - 点击浏览器不同标签，验证侧边栏会自动滚动定位。
