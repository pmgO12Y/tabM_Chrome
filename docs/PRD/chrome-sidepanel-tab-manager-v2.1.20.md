# Chrome 侧边栏标签管理插件 PRD v2.1.20

## Executive Summary
- 本版本在 `v2.1.19` 的基础上，移除了网页组标题按钮的按压缩放动画，改为组内容展开时的轻量展开动画。
- 目标是让交互反馈更符合用户预期，并继续尽量控制性能开销。

## Problem Statement
- `v2.1.19` 加入的是“点击组头按钮时缩一下”的动画，但用户实际要的是“组展开时”的展开动画。
- 当前需要删除错误方向的动画，并替换为更贴近展开行为本身的视觉反馈。

## Target Users
- 高频使用网页组展开/收起，并希望看到更符合心智模型的展开反馈的用户。

## Solution Overview
- 删除 `group-row` 上的按压缩放动画：
  - 去掉 `transform` 过渡
  - 去掉 `:active` 缩放样式
- 在 `group-block__body` 上增加轻量展开动画：
  - 仅在展开挂载时执行
  - 使用 `opacity + scaleY` 的短时动画表现“内容展开”
- 保留 reduced motion 兼容：
  - 当系统偏好减少动画时，关闭该展开动画

## Success Metrics
- 点击网页组时，不再出现“组头按钮缩一下”的错误反馈。
- 展开网页组时，内容区会出现轻量展开过渡。
- 动画仅作用于展开中的组内容区，不引入重型动画和明显性能负担。

## User Stories / Acceptance Criteria
- 作为用户，我希望网页组展开时有展开反馈，而不是组头按钮自己缩一下。
- 作为用户，我希望这个动画足够轻，不要影响侧边栏性能和响应速度。

## Out of Scope
- 本版本不为窗口展开增加同类动画。
- 本版本不引入 JS 动画控制，不改变展开/收起业务逻辑。

## Risks / Dependencies
- 由于当前结构在收起时直接卸载组内容，本版本只保证“展开动画”，不包含对称的收起动画。
- 为控制性能，本次继续只使用 CSS 动画，不改布局计算逻辑。

## Open Questions
