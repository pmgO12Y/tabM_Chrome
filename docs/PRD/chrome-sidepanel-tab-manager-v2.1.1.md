# Chrome 侧边栏标签管理插件 PRD v2.1.1

## Executive Summary
- 本版本在 `v2.1.0` 原生编组功能基础上，强化编组的整体视觉表现。
- 目标是让已编组标签按截图那样显示为“编组头 + 整组边框内容区”，而不是只有一条带颜色的编组头。

## Problem Statement
- `v2.1.0` 已经接入了 Chrome 原生编组数据和折叠联动，但视觉上仍偏“单条编组头”。
- 用户明确要求像截图那样，用带颜色的框把整个组框起来。

## Target Users
- 依赖 Chrome 原生编组管理大量标签，并希望侧边栏编组层级更清晰的用户。

## Solution Overview
- 将编组从“平铺标题行”改为“编组容器”渲染：
  - 上方为带原生颜色的编组头
  - 下方为同色边框包裹的编组内容区
- 编组内标签渲染在内容区内，未编组标签继续保持原样。
- 保持原有编组数据同步和展开/收起联动逻辑不变，让用户可以直接去 Chrome 里测试新建/删除编组联动。

## Success Metrics
- 已编组标签在侧边栏中显示为完整编组块，而不是单独一条标题。
- 编组头颜色与边框颜色保持同一 Chrome 原生颜色体系。
- Chrome 中新建/删除编组后，侧边栏仍能正确联动显示变化。

## User Stories / Acceptance Criteria
- 作为用户，我希望编组在侧边栏里像截图一样被整体框起来。
- 作为用户，我希望编组头和编组内容区是同一组颜色体系。
- 作为用户，我希望继续在 Chrome 顶部新建/删除编组并观察侧边栏联动。

## Out of Scope
- 本版本不新增创建编组、改名、改颜色功能。
- 本版本不改变现有编组折叠联动和标签激活逻辑。

## Risks / Dependencies
- 编组容器化后，样式层需要依赖“编组头后面紧跟该编组标签”的渲染顺序。

## Open Questions
- 当前无新的产品决策问题。
