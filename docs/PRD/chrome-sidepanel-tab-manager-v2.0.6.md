# Chrome 侧边栏标签管理插件 PRD v2.0.6

## Executive Summary
- 本版本在 `v2.0.5` 的悬浮高亮基础上，为当前窗口中的当前选中标签增加左侧竖条指示。
- 目标是让“当前选中目标”更像常见列表里的选中态，强化当前位置识别。

## Problem Statement
- 当前版本虽然已经有浅蓝底色和多层高亮，但“当前窗口中的当前选中标签”仍主要依赖整行底色表达。
- 对用户来说，这种表现已经够用，但缺少一个更明确、更稳定的“选中锚点”。
- 需要增加一个左侧竖着的小矩形，让当前选中状态更直观，同时不破坏已有浅蓝背景和整体极简布局。

## Target Users
- 希望当前标签更容易一眼识别的用户。
- 习惯侧边栏、导航栏或列表中“左侧竖条选中态”的用户。

## Solution Overview
- 为 `tab-row--current-active` 增加左侧贴边的竖向矩形指示条。
- 保留现有浅蓝背景高亮，不改成纯竖条模式。
- 继续只对当前窗口中的当前活动标签显示该竖条：
  - 当前标签：浅蓝底 + 左侧竖条
  - 其他窗口活动标签：弱高亮，无竖条
  - 普通标签：无竖条
- 通过 CSS 伪元素实现，不新增 DOM 节点，不修改逻辑判定。

## Success Metrics
- 当前窗口中的当前选中标签左侧能看到明显但克制的竖条。
- 竖条会跟随当前标签切换而移动。
- 不影响现有高亮层级、自动定位和悬浮反馈。

## User Stories / Acceptance Criteria
- 作为用户，我应能在当前标签左侧看到一个竖着的小矩形选中指示。
- 作为用户，当前标签仍保留整行浅蓝底色，不是只剩竖条。
- 作为用户，其他窗口的活动标签不应出现这个竖条。
- 作为用户，鼠标悬浮到当前标签时，竖条不消失，整体仍保持当前选中态。

## Out of Scope
- 本版本不改变当前网页判定逻辑。
- 本版本不改变滚动或高亮规则。
- 本版本不改变窗口标题行样式。

## Risks / Dependencies
- 这是纯样式增强，不依赖后台逻辑变更。
- 若后续调整标签行左右内边距，需要同步检查竖条与内容区的间距是否仍协调。

## Open Questions
- 如果后续还想让当前标签更有层级感，可以考虑进一步统一“竖条 + 底色 + 当前窗口标记”的整体语言，但本版本先保持最小改动。
