# Chrome 侧边栏标签管理插件 PRD v1.0.9

## 1. Executive Summary
开发一款运行在 Chrome 侧边栏中的标签管理插件，面向 300+ 标签页的重度用户场景，提供按窗口浏览、快速搜索、切换、关闭和固定等基础能力。首版以高性能、低内存、模块化扩展为核心，优先解决“浏览器标签过多导致定位和切换效率下降”的问题，并将侧边栏中的网页图标展示策略收敛为“与 Chrome 顶部标签栏当前显示一致优先”。

## 2. Problem Statement
- 多窗口、多任务并行的 Chrome 重度用户，需要在大量标签中快速定位目标页面。
- 原生横向标签栏难以同时表达窗口上下文和标签层级，切换成本高。
- 当标签被休眠类插件替换为 `chrome-extension://...` 页面时，若侧边栏强行还原原网页图标，会与 Chrome 顶部标签栏此刻看到的图标不一致，增加识别混乱。

## 3. Target Users
- 主用户：100-300+ 标签的重度用户。
- 次用户：希望通过侧边栏获得更稳定浏览体验的普通用户。
- 典型任务：按窗口浏览、搜索定位、快速切换、关闭、固定标签。

## 4. Strategic Context
- Chrome Side Panel 已提供稳定承载容器。
- 首版聚焦高频操作和真实体感性能，而不是复杂整理能力。
- 图标展示应优先服务“当前标签状态识别”，而不是推断历史页面来源。

## 5. Solution Overview
### 产品边界
- 平台：Chrome Desktop。
- 技术基线：Manifest V3 + Side Panel API。
- 不兼容 Firefox / Edge，不提供 popup 或 new tab 版本。

### MVP 能力
- 按窗口展示标签列表，支持展开/折叠。
- 搜索标题与 URL。
- 切换标签、关闭标签、固定/取消固定。
- 查看当前窗口或全部窗口。
- 首屏未就绪时展示加载动画和标签加载进度。
- 浏览器当前激活标签变化时，侧边栏自动滚动定位到对应项。
- 网页图标展示策略以“与 Chrome 顶部标签栏当前显示一致”为优先。

### 图标展示约束
- 普通 `http/https/file` 标签继续优先使用 Chrome `_favicon` 代理，保持与当前网页一致。
- `chrome-extension://...`、`chrome://...`、`data:`、`blob:` 等内部页或插件页，不再尝试还原“原网页图标”。
- 对内部页或插件页，优先使用 `chrome.tabs.Tab.favIconUrl` 当前提供的安全 favicon。
- 若当前标签没有安全 favicon，则回退为标题首字母。
- 该策略只影响左侧网页图标，不改变标题和 URL 文本。

## 6. Success Metrics
- 300+ 标签场景下首次可交互渲染不高于 500ms。
- 1000 标签场景下若首屏尚未可见，必须显示加载状态与进度。
- 搜索输入到结果刷新不高于 100ms。
- 单个标签事件到 UI 反映不高于 50ms。
- 休眠插件替换页在侧边栏中显示的图标，应与 Chrome 顶部标签栏此刻显示的图标一致或同源降级。

## 7. User Stories / Acceptance Criteria
- 作为用户，我可以按窗口查看标签，以理解当前工作上下文。
- 作为用户，我可以搜索标题或 URL，以更快定位目标标签。
- 作为用户，我可以直接切换、关闭或固定一个标签，而不必回到顶部标签栏。
- 作为用户，当标签被休眠插件替换后，侧边栏显示的图标应跟随 Chrome 当前顶部标签图标，而不是强行显示原网页图标。
- 验收标准：
  - `https`、`http`、`file` 标签继续显示对应网页 favicon。
  - 休眠插件替换页如果当前标签有扩展页 favicon，应优先显示该 favicon。
  - 休眠插件替换页即使 URL 参数中带原网页地址，也不再强行还原原网页 favicon。
  - 当前安全 favicon 不可用时，回退为首字母。
  - 标题和 URL 文本不因该策略发生变化。

## 8. Out of Scope
- 保存和恢复整组标签页。
- 多选和批量整理。
- 智能分类、规则引擎或 AI 功能。
- 账号同步与跨设备同步。
- 针对单个休眠插件做“原网页图标还原”适配。

## 9. Risks / Dependencies
- 依赖 Chrome Side Panel API、Manifest V3 service worker、`chrome.tabs` 和 `chrome.storage`。
- Chrome 顶部标签栏的最终绘制仍由浏览器控制；侧边栏只能基于 `pageUrl` 与 `favIconUrl` 尽量对齐，不能控制浏览器自身图标渲染。

## 10. Open Questions
- 后续是否需要让普通网页标签也改成直接跟随 `tab.favIconUrl`，进一步逼近顶部标签栏的像素级一致。
- 是否需要在 UI 中增加“休眠标签”弱提示，帮助用户理解为什么它显示的是插件页图标。

## 11. Verification Plan
- 自动验证：
  - 单元测试覆盖普通网页 favicon 代理、扩展页 favicon 保留、带原网页参数的休眠页不做还原、无安全 favicon 时回退为空。
  - 类型检查确保 `TabRecord` 和归一化逻辑保持兼容。
  - 构建产物必须可生成 `background` 与 `sidepanel` 两个入口。
- 手工验证：
  - 验证正常网页标签的图标与 Chrome 标签栏一致。
  - 验证 `chrome://`、`chrome-extension://`、`file://` 标签行为符合预期。
  - 验证休眠插件替换页存在原网页参数时，侧边栏仍跟随当前顶部标签图标，而不还原原网页 favicon。
  - 验证当前安全 favicon 缺失时，侧边栏回退为首字母。
  - 验证标题和 URL 文本保持不变。
