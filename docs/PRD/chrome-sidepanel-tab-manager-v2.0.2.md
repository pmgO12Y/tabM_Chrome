# Chrome 侧边栏标签管理插件 PRD v2.0.2

## Executive Summary
- 本版本在 `v2.0.1` 的极简定位器基础上，修正窗口分组排序策略。
- 目标是让侧边栏窗口顺序固定，不再因为当前聚焦窗口变化而跳动。

## Problem Statement
- 之前的窗口分组顺序会把当前聚焦窗口自动置顶。
- 这会导致用户在不同 Chrome 窗口之间切换网页时，侧边栏里的窗口分组顺序不断变化。
- 对“极简定位器”场景来说，窗口位置跳动会增加认知负担，也不符合“窗口排序固定，不要乱动”的要求。

## Target Users
- 需要在多个 Chrome 窗口之间稳定定位网页的用户。
- 更在意列表顺序稳定性，而不是当前窗口优先显示的用户。

## Solution Overview
- 将底层 `windowOrder` 的生成规则改为按 `windowId` 固定升序。
- 保留 `focusedWindowId`，但它只用于表达当前窗口状态，不再参与窗口排序。
- 侧边栏窗口标题改为稳定文案 `窗口 1 / 窗口 2 / ...`，不再随着当前焦点切换成“当前窗口”。
- 当前窗口仍保留轻量“当前”标记与高亮样式，但始终停留在原位置。
- 当前窗口自动展开、当前标签自动定位与高亮功能继续保留。

## Success Metrics
- 在不同 Chrome 窗口之间切换网页时，侧边栏窗口分组位置保持不变。
- 当前窗口仍然可以被正确标识出来，但不会触发重排。
- 不影响现有点击跳转、自动定位、图标显示与构建验证。

## User Stories / Acceptance Criteria
- 作为用户，我在侧边栏看到的窗口分组顺序应保持固定，不因焦点变化而移动。
- 作为用户，我切换到另一个 Chrome 窗口中的网页时，侧边栏只更新“当前”状态，不应把该窗口挪到最前面。
- 作为用户，我仍然可以一眼看出哪个窗口是当前窗口。
- 作为用户，如果当前窗口此前被我收起，切换到该窗口中的网页后，侧边栏应自动展开它，但不改变排序。

## Out of Scope
- 本版本不改变标签点击跳转逻辑。
- 本版本不改变 favicon 规则。
- 本版本不改变窗口分组的展开/收起交互方式。

## Risks / Dependencies
- 当前采用 `windowId` 固定升序，不等同于用户肉眼看到的打开先后顺序；这是本版本明确接受的规则。
- 若未来用户希望“按首次出现顺序固定”，需再单独调整底层排序语义。

## Open Questions
- 如果后续需要支持用户自定义窗口排序方式，应考虑把排序策略显式抽象成一个单独配置项，而不是再次把“当前窗口优先”混回默认行为。
