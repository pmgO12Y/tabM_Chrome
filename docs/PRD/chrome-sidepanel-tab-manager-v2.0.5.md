# Chrome 侧边栏标签管理插件 PRD v2.0.5

## Executive Summary
- 本版本在 `v2.0.4` 的平滑定位体验基础上，补充鼠标悬浮反馈。
- 目标是让窗口行和标签行在鼠标悬浮时出现浅色高亮，提升可感知性，但不破坏当前已有的活动状态层级。

## Problem Statement
- 当前列表虽然已经支持当前标签高亮、其他窗口活动标签弱高亮和当前窗口标记，但鼠标悬浮时没有额外反馈。
- 对用户来说，这会降低可点击区域的可感知性，尤其是窗口行和普通标签行。
- 需要补一个很轻的悬浮态，让鼠标停留时能看到淡淡底色，但不能盖掉原本更重要的活动高亮。

## Target Users
- 希望侧边栏在日常鼠标操作时更直观的用户。
- 需要快速判断“当前鼠标停在哪一行”的用户。

## Solution Overview
- 给窗口行和普通标签行增加统一的浅色 `hover` 背景。
- 对当前窗口活动标签和其他窗口活动标签，在悬浮时分别使用略强一点的对应背景色，保持视觉层级：
  - 当前窗口活动标签：最强高亮
  - 其他窗口活动标签：次强高亮
  - 普通悬浮：最浅高亮
- 不引入动画、不引入阴影、不改变现有结构和逻辑。

## Success Metrics
- 鼠标停在窗口行或标签行上时，会出现浅色高亮反馈。
- 当前活动标签和其他窗口活动标签在悬浮时仍保留明显层级差异。
- 不影响现有自动定位、窗口排序和标签高亮行为。

## User Stories / Acceptance Criteria
- 作为用户，我把鼠标放到窗口行上时，应看到浅色悬浮高亮。
- 作为用户，我把鼠标放到普通标签行上时，也应看到浅色悬浮高亮。
- 作为用户，我把鼠标放到当前活动标签上时，它应继续比普通悬浮更明显。
- 作为用户，我把鼠标放到其他窗口的活动标签上时，它也应比普通标签稍强，但仍弱于当前活动标签。

## Out of Scope
- 本版本不改变滚动逻辑。
- 本版本不改变当前网页判定逻辑。
- 本版本不改变图标或窗口排序逻辑。

## Risks / Dependencies
- 这是纯样式增强，不涉及业务逻辑风险。
- 若后续继续增加更多状态色，需要注意避免悬浮态与活动态颜色过于接近。

## Open Questions
- 如果后续用户希望更明显的悬浮感，可以再考虑加入极短的颜色过渡；本版本先保持无动画。
