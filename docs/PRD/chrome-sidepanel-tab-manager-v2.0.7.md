# Chrome 侧边栏标签管理插件 PRD v2.0.7

## Executive Summary
- 本版本在 `v2.0.6` 的当前标签竖条选中态基础上，修复侧边栏缩窄后的布局问题。
- 目标是让侧边栏在更窄宽度下仍能完整显示窗口标题行的右侧信息，不再出现横向滚动和信息被挤没。

## Problem Statement
- 当前实现把侧边栏最小宽度硬写成 `300px`，导致用户缩窄面板时出现横向溢出和横向滚动。
- 同时，窗口标题行采用单行横向布局，窄宽度下右侧的“当前 / 数量”会被标题挤掉。
- 这会让缩窄后的侧边栏既难看又不实用，关键信息无法稳定保留。

## Target Users
- 习惯把 Chrome 侧边栏缩到较窄宽度使用的用户。
- 希望在窄宽度下仍然能看到窗口状态和数量信息的用户。

## Solution Overview
- 删除 `sidepanel.html` 和 `styles.css` 中的 `min-width: 300px` 硬限制。
- 给窗口标题行增加 `.window-row__meta` 包裹层，把“当前”标记和数量放到同一组。
- 将窗口标题行布局从简单横向 `flex` 改成响应式 `grid`：
  - 正常宽度下仍保持单行展示
  - 窄宽度下自动切成两行
  - 标题继续单行省略
  - 元信息换到下一行右对齐显示
- 给滚动容器加 `overflow-x: hidden`，避免底部横向滚动条。

## Success Metrics
- 把侧边栏缩窄到比之前截图更窄时，“当前 / 数量”等右侧信息仍可见。
- 缩窄后不再出现底部横向滚动条。
- 正常宽度下，布局仍保持原来的一行显示习惯。
- 不影响当前高亮、弱高亮、竖条、悬浮态和自动定位逻辑。

## User Stories / Acceptance Criteria
- 作为用户，我缩窄侧边栏后，窗口标题右侧的信息不应消失。
- 作为用户，缩窄后窗口标题可以省略，但“当前 / 数量”应该继续显示。
- 作为用户，缩窄后不应再看到横向滚动条。
- 作为用户，普通宽度下的窗口标题行布局不应明显变形。

## Out of Scope
- 本版本不改变标签行布局策略。
- 本版本不改变窗口排序和自动定位逻辑。
- 本版本不改变当前标签高亮、竖条和悬浮层级。

## Risks / Dependencies
- 窄宽响应式行为依赖浏览器正确应用 `@media` 断点和 CSS Grid。
- 若后续窗口标题区域继续增加更多信息项，需要同步复查两行布局是否仍可容纳。

## Open Questions
- 如果未来需要更极端的窄宽模式，可以再单独设计“超窄模式”，例如隐藏部分次要信息；本版本先以“信息不丢”为第一目标。
