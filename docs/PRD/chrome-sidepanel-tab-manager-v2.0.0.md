# Chrome 侧边栏标签管理插件 PRD v2.0.0

## Executive Summary
- 本版本将插件从“侧边栏标签管理器”重构为“侧边栏极简定位器”。
- 只保留两项核心能力：
  - 侧边栏按窗口分组显示网页图标与网页标题
  - 点击侧边栏标签切换到对应网页；在 Chrome 中切换网页时，侧边栏自动定位并高亮当前项

## Problem Statement
- 旧版本同时包含搜索、范围切换、关闭标签、固定标签、偏好持久化、本地预览等多项功能。
- 对当前需求来说，这些能力都会增加界面复杂度和代码维护成本，也会让“快速定位网页”这个核心目的被稀释。
- 需要把插件收缩为一个单一职责工具，只做显示、定位和跳转。

## Target Users
- 打开很多标签页，但不希望在侧边栏中再做复杂管理操作的用户。
- 更关注“快速找到并切换网页”的用户。

## Solution Overview
- 后台脚本继续监听 Chrome 标签与窗口变化，维护实时标签快照。
- 侧边栏改为极简分组列表：
  - 显示窗口标题
  - 显示网页图标和标题
  - 点击标签执行切换
- 当前激活网页在侧边栏中保持高亮，并在状态变化时自动滚动到可见区域。
- 若当前网页所在窗口此前被手动收起，则在自动定位前先自动展开该窗口分组。
- 收起状态只保留在当前侧边栏会话中，不再写入 `chrome.storage`。

## Success Metrics
- 用户能在侧边栏中看到所有打开网页的图标与标题。
- 用户点击任意侧边栏标签后，Chrome 能切换到对应网页并聚焦对应窗口。
- 用户在 Chrome 中切换网页时，侧边栏会自动定位并高亮当前标签。
- 插件界面中不再出现搜索、关闭、固定、范围切换、预览模式等旧功能。

## User Stories / Acceptance Criteria
- 作为用户，我可以在侧边栏中按窗口查看所有网页标签，并看到每个标签的图标与标题。
- 作为用户，我点击侧边栏中的标签后，浏览器会自动切换到对应网页。
- 作为用户，我直接在 Chrome 中点击其他网页标签时，侧边栏会自动滚动到对应项并高亮显示。
- 作为用户，如果我手动收起某个窗口分组，当该窗口中的网页再次成为当前网页时，侧边栏会自动展开该分组。
- 作为用户，重新打开侧边栏后，所有窗口分组默认展开，不继承上一次收起状态。

## Out of Scope
- 本版本不再支持标签搜索。
- 本版本不再支持关闭标签、固定标签、切换“当前窗口/全部窗口”范围。
- 本版本不再支持偏好持久化和本地预览模式。
- 本版本不改动网页图标的兜底策略，继续保留原有 favicon 回退链路。

## Risks / Dependencies
- 侧边栏点击切换仍依赖后台脚本与 `chrome.tabs`、`chrome.windows` API 正常工作。
- 当前实现仍会保留 `TabRecord` 中部分非展示字段，虽然不再参与界面功能，但后续若继续精简，可再做数据结构收缩。

## Open Questions
- 若后续需要“极简模式”和“管理模式”并存，应考虑拆分为显式模式开关，而不是继续在同一默认界面里叠加功能。
