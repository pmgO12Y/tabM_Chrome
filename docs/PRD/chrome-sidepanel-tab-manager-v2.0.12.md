# Chrome 侧边栏标签管理插件 PRD v2.0.12

## Executive Summary
- 本版本纠正此前对窄宽问题的定位偏差，重点改为网页标题行的收缩链路。
- 目标是让长网页标题只能在自己的行内省略为 `...`，不能再反过来撑宽整个侧边栏，进而挤掉窗口行右侧信息。

## Problem Statement
- 用户明确指出此前反复调整的是窗口标题，但真正有问题的是网页标题。
- 如果网页标题所在的列表容器、列表项或标签按钮保留了默认最小宽度，长标题就可能把整列撑宽。
- 一旦整列实际宽度超过可视区域，顶部窗口行右侧的“当前 / 数量”就会看起来像消失了。

## Target Users
- 会把侧边栏缩到较窄甚至最窄状态的用户。
- 需要同时看到窗口右侧状态信息和下方标签列表的用户。

## Solution Overview
- 让列表容器、列表项和标签按钮都显式允许继续缩小。
- 为 `.stack-list` 增加 `grid-template-columns: minmax(0, 1fr)`，避免隐式网格列被长内容撑宽。
- 为 `.virtual-list`、`.stack-list__item`、`.window-row`、`.tab-row` 增加 `min-width: 0`，打通网页标题的收缩链路。
- 保持网页标题现有的单行省略策略，让长标题在自己这一行里显示 `...`。

## Success Metrics
- 侧边栏缩窄时，网页标题会在自己的行内被省略，而不是撑宽整个列表。
- 窗口行右侧“当前 / 数量”不会再因为下方长网页标题而看起来消失。
- 不引入新的横向滚动条。

## User Stories / Acceptance Criteria
- 作为用户，我缩窄侧边栏时，网页标题应该先变短并显示 `...`。
- 作为用户，我不希望下方长网页标题把顶部窗口行右侧信息挤出可视区。
- 作为用户，我希望整个列表在窄宽下仍保持稳定。

## Out of Scope
- 本版本不改窗口排序、自动定位和高亮逻辑。
- 本版本不改消息协议和数据结构。

## Risks / Dependencies
- 如果未来标签行继续增加更多固定宽度元素，仍需要保持这一整条收缩链路可用。
- 极窄宽度下，网页标题可见文本会更短，这是主动选择的显示优先级结果。

## Open Questions
- 当前无新的产品决策问题；若后续仍有特殊页面标题导致问题，可再单独针对标签行元素占宽做约束。
