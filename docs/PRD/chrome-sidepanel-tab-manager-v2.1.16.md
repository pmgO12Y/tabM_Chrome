# Chrome 侧边栏标签管理插件 PRD v2.1.16

## Executive Summary
- 本版本在 `v2.1.15` 的基础上，继续修复侧边栏下方网页组/窗口展开收起时仍会跳动的问题。
- 目标是补足“锚定计算正确但滚动容器底部没有足够可滚空间”的场景，让标题固定策略在靠近底部时也能真正生效。

## Problem Statement
- `v2.1.15` 已经实现了“点击前记录标题位置，更新后回补 scrollTop”的锚定逻辑。
- 但当网页组或窗口靠近侧边栏底部时，滚动容器可能已经到达可滚动上限；即使计算出了正确的补偿值，也会因为没有剩余滚动空间而被浏览器截断，导致标题仍然跳动。

## Target Users
- 所有在侧边栏下半区操作网页组或窗口展开/收起的用户。

## Solution Overview
- 在 `VirtualizedWindowList` 中新增“动态底部 spacer”：
  - 仅在手动展开/收起导致目标补偿超出当前最大 `scrollTop` 时，临时在列表底部补一段空白占位
  - 用这段额外空间让锚定滚动补偿真正有地方可退
- 保留 `v2.1.15` 的标题锚定逻辑，但在应用补偿前先计算是否需要底部空间：
  - 若当前最大可滚动距离不足，则先提升底部 spacer 高度
  - 下一轮布局再执行真正的滚动补偿
- 当用户后续向上滚动，已经离开这段人工补出来的尾部空间后，自动移除 spacer，避免长期留下大块底部空白
- 保持原有“手动 toggle 抑制一次自动活动标签定位”的规则不变

## Success Metrics
- 网页组或窗口位于侧边栏底部附近时，点击展开/收起后，标题位置明显更稳定。
- 只有在确实不够滚动空间时才出现额外底部空白，占位会在安全时机自动释放。
- 真实切换标签时的自动定位不受影响。

## User Stories / Acceptance Criteria
- 作为用户，我希望底部附近的网页组展开/收起时，不再因为滚动空间不够而乱跳。
- 作为用户，我希望额外补出来的空白不是长期常驻，而是只在需要时临时出现。

## Out of Scope
- 本版本不改组外观颜色、边框和动画。
- 本版本不改后台数据同步逻辑。

## Risks / Dependencies
- 动态 spacer 依赖前端测量滚动容器高度和当前位置，必须在布局阶段计算。
- 如果后续又引入新的自动滚动逻辑，需要继续遵守“手动交互优先”的顺序约束。
