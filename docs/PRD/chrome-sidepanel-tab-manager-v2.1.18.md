# Chrome 侧边栏标签管理插件 PRD v2.1.18

## Executive Summary
- 本版本在 `v2.1.17` 的基础上，调整窗口标题规则，并压缩网页组标题栏高度。
- 目标是让窗口行同时保留“窗口序号”和“当前活动标签标题”，并让组标题栏更紧凑，减少占据的竖向空间。

## Problem Statement
- `v2.1.17` 将窗口标题完全改成了当前活动标签标题，但用户仍然希望保留“窗口 1/2/3/4”这种序号识别。
- 当前网页组标题栏仍偏高，用户希望明显缩短标题栏高度。

## Target Users
- 需要同时识别窗口序号和窗口当前内容，并希望侧边栏分组头更紧凑的用户。

## Solution Overview
- 在 `selectors.ts` 中将窗口标题改为：
  - 有活动标签标题时：`窗口 N - 活动标签标题`
  - 没有可用活动标签标题时：`窗口 N`
- 保留 `v2.1.17` 中“未命名编组直接留空”的行为不变。
- 在 `styles.css` 中压缩组标题栏：
  - 缩小上下 padding
  - 缩小标题、数量、箭头字号
  - 缩小右侧 meta 间距

## Success Metrics
- 窗口标题重新包含“窗口 1/2/3/4”序号，同时继续显示当前活动标签标题。
- 网页组标题栏明显更瘦，占用高度显著下降。
- 组折叠、窗口折叠、数量显示和点击行为不受影响。

## User Stories / Acceptance Criteria
- 作为用户，我希望窗口行标题既能看出“这是第几个窗口”，也能看出这个窗口当前活动页是什么。
- 作为用户，我希望网页组标题栏不要太厚，占更少的竖向空间。

## Out of Scope
- 本版本不改窗口和编组的联动逻辑。
- 本版本不调整组边框颜色和整体布局结构。

## Risks / Dependencies
- 组标题栏高度明显下降后，点击区域也会随之变小；本次按“视觉接近减半但仍能正常点按”的标准收缩，而不是机械砍到一半。
