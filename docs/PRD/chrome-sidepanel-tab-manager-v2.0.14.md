# Chrome 侧边栏标签管理插件 PRD v2.0.14

## Executive Summary
- 本版本强化窗口标题行的分组展示效果。
- 目标是让所有窗口标题行都更像清晰的分组条，和下方网页标题行拉开视觉层级，同时保留当前窗口更强一点的状态提示。

## Problem Statement
- 当前窗口标题行虽然有当前窗口加粗和轻微悬浮反馈，但整体层级仍偏普通。
- 用户要求“侧边栏的窗口标题行重点展示一下”，说明现有分组标题不够醒目，不利于快速扫视分组结构。

## Target Users
- 标签较多、需要快速扫视窗口分组结构的用户。
- 习惯通过侧边栏快速定位不同窗口分组的用户。

## Solution Overview
- 为所有窗口标题行增加统一的浅色底，做成更明显的分组条。
- 适度增加上下留白和基础字重，让窗口标题行与网页标题行形成稳定层级差。
- 当前窗口继续复用现有 focused 语义，在统一浅底色基础上再略增强底色和字重。
- 不新增边框、阴影、动画等额外效果。

## Success Metrics
- 所有窗口标题行都能一眼看出是分组标题，而不是普通列表行。
- 当前窗口标题行比其他窗口稍强，但不会喧宾夺主。
- 不影响展开/收起、自动定位、窄宽标题省略和右侧“当前 / 数量”显示。

## User Stories / Acceptance Criteria
- 作为用户，我希望所有窗口标题行更醒目，方便快速识别分组。
- 作为用户，我希望当前窗口标题行比其他窗口稍微更突出。
- 作为用户，我不希望为了强调窗口标题行而加入边框、阴影或动画。

## Out of Scope
- 本版本不改窗口排序、自动定位、高亮逻辑和交互逻辑。
- 本版本不改网页标题的省略策略。

## Risks / Dependencies
- 如果后续窗口标题行继续增加更多操作控件，需要再次确认浅底色和留白是否仍协调。
