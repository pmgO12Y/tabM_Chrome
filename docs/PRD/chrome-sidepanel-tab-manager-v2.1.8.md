# Chrome 侧边栏标签管理插件 PRD v2.1.8

## Executive Summary
- 本版本在 `v2.1.7` 的基础上，修正网页组内部标签行与组外普通标签行的横向对齐问题。
- 目标是让组内标签的 favicon 和标题文字，与组外标签保持同一条视觉起点。

## Problem Statement
- 当前网页组内部的标签行，除了自身的左内边距，还额外受到组容器左侧外边距和边框的影响。
- 这会导致组内图标和文字整体向右偏，和组外标签看起来不在一条竖线上。

## Target Users
- 所有使用侧边栏查看分组与未分组标签的用户。

## Solution Overview
- 调整 `src/sidepanel/styles.css` 中组内标签行的左内边距。
- 通过抵消组容器带来的额外横向偏移，让组内标签的内容起点与组外普通标签保持一致。

## Success Metrics
- 网页组内的图标与标题文字，在视觉上与组外普通标签对齐。
- 不改变现有网页组外框、颜色和交互逻辑。

## User Stories / Acceptance Criteria
- 作为用户，我希望网页组内部的标签不要比组外标签明显更靠右。
- 作为用户，我希望浏览侧边栏时，不同标签行的图标和文字看起来整齐对齐。

## Out of Scope
- 本版本不修改网页组交互逻辑。
- 本版本不调整颜色、字号或动画策略。

## Risks / Dependencies
- 本版本属于样式级 patch，需要通过构建确认未引入 CSS 或打包异常。

## Open Questions
- 当前无新的产品决策问题。
