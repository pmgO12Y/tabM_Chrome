# Chrome 侧边栏标签管理插件 PRD v2.1.9

## Executive Summary
- 本版本在 `v2.1.8` 的基础上，调整网页组标题栏的颜色表现。
- 目标是让组标题栏底色更接近 Chrome 原生组色，并将标题文字、数量和箭头统一改为白色。

## Problem Statement
- 当前网页组标题栏使用的是偏浅的自定义背景色，和 Chrome 顶部原生标签组颜色不够接近。
- 当前标题栏文字仍按深色方案渲染，和用户期望的白色标题栏文字不一致。
- 如果只改文字色而不拆分边框色变量，会导致组边框也被错误改成白色。

## Target Users
- 所有在侧边栏中查看和操作网页组的用户。

## Solution Overview
- 调整 `src/sidepanel/styles.css` 中各个网页组颜色变量，使标题栏颜色改为更接近 Chrome 原生的高饱和组色。
- 将网页组标题栏文字色统一改为白色。
- 新增独立的 `outline` 颜色变量，保证网页组边框继续使用组色本身，不会被文字白色误带偏。

## Success Metrics
- 网页组标题栏的背景色更接近 Chrome 原生标签组颜色。
- 标题、数量和箭头统一为白色。
- 网页组边框颜色保持稳定，不会因为标题文字改白而失真。

## User Stories / Acceptance Criteria
- 作为用户，我希望侧边栏里的网页组标题颜色看起来更像 Chrome 自己顶部标签组的颜色。
- 作为用户，我希望网页组标题、数量和箭头统一为白色，视觉上更干净。
- 作为用户，我希望网页组边框不要因为文字改白而一起变白。

## Out of Scope
- 本版本不调整网页组折叠逻辑。
- 本版本不修改组内标签排版与交互行为。

## Risks / Dependencies
- Chromium 公开资料里可直接查到的是颜色枚举，不是完整前端 hex 对照表；本次颜色值是基于 Chromium 原生色系做的贴近式映射。
- 本版本属于样式级 patch，需要通过构建确认未引入 CSS 或打包异常。
