# Chrome 侧边栏标签管理插件 PRD v2.1.17

## Executive Summary
- 本版本在 `v2.1.16` 的基础上，去掉了未命名编组的默认文案，并取消窗口行的“窗口 1/2/3/4”编号标题。
- 目标是让空编组标题直接留空，同时让窗口行显示更有实际意义的“当前活动标签标题”。

## Problem Statement
- 当前未命名 Chrome 编组会在归一化时被强行改成“未命名编组”，这不符合用户希望“直接空白”的展示要求。
- 当前窗口行标题由代码按顺序生成“窗口 1/2/3/4”，信息量低，也不符合用户要求。

## Target Users
- 使用 Chrome 侧边栏管理多个窗口和网页组，并更在意真实内容而不是占位文案的用户。

## Solution Overview
- 在 `normalizeGroup.ts` 中移除 `"未命名编组"` 回退文案：
  - 继续对 `group.title` 做 `trim()`
  - 结果为空时保留为 `""`
- 在 `selectors.ts` 中取消 `窗口 ${index + 1}` 标题生成：
  - 改为从该窗口 tabs 中找到 `active === true` 的标签
  - 使用其 `title.trim()` 作为窗口标题
  - 若没有可用活动标签标题，则窗口标题保留为 `""`
- 渲染层不额外补文案，直接显示标题字段原值

## Success Metrics
- 未命名编组在侧边栏中不再显示“未命名编组”，而是空白标题。
- 窗口行不再显示“窗口 1/2/3/4”，而是显示该窗口当前活动标签标题。
- 若窗口当前活动标签标题不可用，则保持空白，不补默认文案。

## User Stories / Acceptance Criteria
- 作为用户，我希望未命名编组在侧边栏里直接留空，而不是显示“未命名编组”。
- 作为用户，我希望窗口行显示当前活动标签标题，而不是“窗口 1/2/3/4”。
- 作为用户，我希望这些标题变更不影响窗口/编组的折叠、数量、颜色和点击行为。

## Out of Scope
- 本版本不改窗口和组头的布局结构。
- 本版本不新增窗口命名能力，也不从 Chrome 读取不存在的窗口原生标题。

## Risks / Dependencies
- Chrome 普通窗口没有专门的原生名称字段，因此窗口标题只能从现有标签信息中推导。
- 当某个窗口拿不到活动标签标题时，显示为空白是本次明确选择，不再做回退编号。
