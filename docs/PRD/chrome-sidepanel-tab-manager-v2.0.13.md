# Chrome 侧边栏标签管理插件 PRD v2.0.13

## Executive Summary
- 本版本修复 `v2.0.12` 对窗口标题的误伤。
- 目标是在保留网页标题可继续收缩并显示 `...` 的前提下，把窗口名正常显示补回来。

## Problem Statement
- `v2.0.12` 为了让标题区域更容易收缩，给 `.window-row__title` 加了 `width: 0` 等过强约束。
- 用户反馈“窗口名再给我补回来”，说明这部分已经超出真实需求，误伤了窗口标题显示。
- 用户真正要压缩的是网页标题，而不是窗口标题。

## Target Users
- 需要同时看到窗口分组名称和网页标题列表的用户。
- 会在窄宽度下使用侧边栏的用户。

## Solution Overview
- 删除窗口标题 `.window-row__title` 上过度的宽度压缩规则。
- 保留窗口标题现有的单行省略能力，但不再强制宽度归零。
- 保留网页标题所在列表的收缩链路，不回退 `v2.0.12` 针对网页标题的修正。

## Success Metrics
- 窗口名恢复正常显示。
- 网页标题仍可在侧边栏缩窄时优先省略为 `...`。
- 不影响窗口右侧状态信息和现有交互逻辑。

## User Stories / Acceptance Criteria
- 作为用户，我希望窗口名恢复显示。
- 作为用户，我希望网页标题继续在空间不足时缩短成 `...`。
- 作为用户，我不希望为了修网页标题而把窗口名一起压没。

## Out of Scope
- 本版本不改窗口排序、自动定位、高亮和交互逻辑。
- 本版本不改消息协议和数据结构。

## Risks / Dependencies
- 若未来继续大幅调整窗口行布局，需要注意不要再次误伤窗口标题。
