# Chrome 侧边栏标签管理插件 PRD v2.1.2

## Executive Summary
- 本版本在 `v2.1.1` 原生编组功能基础上，修复网页组点击联动逐渐变得不稳定的问题。
- 目标是让侧边栏点击编组头时，Chrome 顶部真实编组持续稳定同步，不再出现“前几个能点、后面越来越不稳”的现象。

## Problem Statement
- 当前后台编组同步链路会在同步组时再次回到“按标签同步组”的路径，存在自我回调风险。
- 当前侧边栏会在每次渲染时反复强制展开当前网页所在组，导致用户手动收起后看起来像“点击没生效”。

## Target Users
- 依赖 Chrome 原生标签编组整理大量网页，并通过侧边栏快速切换和折叠编组的用户。

## Solution Overview
- 将后台编组同步拆成两条独立流程：
  - 标签更新时，只在组信息缺失时补拉一次编组元数据
  - 编组创建或更新时，再单独同步该组元数据和组内标签
- 保证后台编组同步不再通过函数自我回调，减少联动排队被拖慢或卡住的风险。
- 将前端自动展开策略改为“一次性自动展开”：
  - 侧边栏启动后，如果当前网页所在组是折叠态，自动展开一次
  - 当前网页切换进新的折叠组时，再自动展开一次
  - 用户手动收起后，不再被同一轮状态刷新立即改回展开

## Success Metrics
- 连续点击多个不同网页组时，后续组仍能正常联动。
- 组内标签发生更新后，后续编组点击仍可继续同步。
- 当前网页所在组在启动时可自动展开一次，便于定位。
- 当前网页所在组被用户手动收起后，不会立刻被程序强制展开。

## User Stories / Acceptance Criteria
- 作为用户，我希望侧边栏连续点击不同编组时都能稳定同步到 Chrome 顶部。
- 作为用户，我希望当前网页第一次定位时能自动展开所在组，但我手动收起后不要被系统抢回来。

## Out of Scope
- 本版本不新增创建编组、改编组名或改编组颜色功能。
- 本版本不改动现有编组视觉样式。

## Risks / Dependencies
- 修复仍依赖 Chrome `tabGroups` 事件与 `tabs` 事件的正常触发顺序。
- 自动展开改为一次性后，首次展开依赖后台 patch 回流完成，真实展开会比纯本地假展开稍晚一步，但逻辑更稳定。
