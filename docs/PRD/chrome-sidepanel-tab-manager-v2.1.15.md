# Chrome 侧边栏标签管理插件 PRD v2.1.15

## Executive Summary
- 本版本在 `v2.1.14` 的基础上，修复侧边栏下方网页组和窗口在展开/收起时标题位置乱跳的问题。
- 目标是让用户手动点击展开/收起后，标题行尽量停留在原来的屏幕位置，不被列表重排和自动定位逻辑抢走滚动位置。

## Problem Statement
- 当前侧边栏在网页组或窗口位于视口下方时，点击展开/收起后会出现明显位置跳动，影响阅读和连续操作体验。
- 根因不是单纯样式，而是列表重排与“自动滚动到当前活动标签”这套逻辑在某些场景下抢同一根滚动条。

## Target Users
- 所有在侧边栏中使用窗口折叠和网页组折叠的用户，尤其是标签较多、需要滚动浏览列表的用户。

## Solution Overview
- 给 `.panel-scroll` 增加显式 `ref`，并传入 `VirtualizedWindowList`，让列表能直接控制实际滚动容器。
- 用户手动点击网页组或窗口标题前，先记录该行在滚动容器中的纵向位置。
- 展开/收起导致列表重排后，在 `useLayoutEffect` 中重新测量该行位置，用前后位置差回补 `scrollTop`，尽量保持标题留在原位。
- 将手动展开/收起视为一次“用户主导交互”，抑制这一轮针对当前活动标签的自动 `scrollIntoView(...)`。
- 对“手动收起导致当前活动标签暂时不在列表里”的场景，也把这次自动定位机会视为已消费，避免后续再次展开时突然跳回当前活动标签。

## Success Metrics
- 网页组位于侧边栏下方时，点击展开/收起后，组标题不再明显上跳或下跳。
- 窗口位于侧边栏下方时，点击展开/收起后，窗口标题也保持相同稳定性。
- 手动展开/收起不会触发多余的自动定位，真实切换当前标签时仍然保留自动定位能力。

## User Stories / Acceptance Criteria
- 作为用户，我希望点击侧边栏下方的网页组展开/收起时，组标题尽量停在原位，而不是自己跑掉。
- 作为用户，我希望窗口和网页组的展开/收起都有一致的稳定体验。
- 作为用户，我希望真正切换网页时，当前标签仍然能继续自动定位，不受这次修复影响。

## Out of Scope
- 本版本不改分组外观样式，不新增动画。
- 本版本不修改后台同步逻辑和对外命令结构。

## Risks / Dependencies
- 网页组展开/收起依赖后台同步回传状态，因此标题位置补偿需要允许一次异步状态更新后再生效。
- 位置补偿依赖 DOM 测量，必须在布局阶段执行，不能延后到普通渲染后逻辑。
