# Chrome 侧边栏标签管理插件 PRD v2.0.8

## Executive Summary
- 本版本在 `v2.0.7` 的窄宽布局修复基础上，收回错误的“双行自适应”方向，改为右侧信息固定锚定。
- 目标是让窗口行右侧的“当前”和“数量”无论侧边栏宽度如何变化，都始终固定显示在最右侧。

## Problem Statement
- `v2.0.7` 虽然尝试解决窄宽下右侧信息被挤掉的问题，但采用了“窄宽时自动换两行”的布局思路。
- 这个方向不符合用户真正需求，因为用户要的是右侧信息作为固定锚点长期保留，而不是在不同宽度下切换展示方式。
- 如果后续右侧还会加入按钮，继续依赖断点和单双行切换会让布局越来越不稳定。

## Target Users
- 需要频繁缩窄 Chrome 侧边栏宽度的用户。
- 希望把窗口右侧区域当成固定信息区或固定动作区使用的用户。

## Solution Overview
- 保持窗口行单行三段结构：左侧箭头、中间标题、右侧信息区。
- 中间标题继续使用省略策略，作为唯一允许被压缩的区域。
- 右侧 `.window-row__meta` 改为固定右对齐、不换行、不收缩。
- 删除 `v2.0.7` 新增的窄宽断点和两行布局逻辑，避免后续继续偏离“右侧固定”的规则。

## Success Metrics
- 缩窄侧边栏后，右侧“当前”和数量仍稳定停在最右侧。
- 继续缩窄时，被优先压缩的是窗口标题，而不是右侧信息区。
- 不再出现因为断点切换导致的信息位置变化。
- 未来右侧加入短按钮时，也能沿用同一固定右侧规则。

## User Stories / Acceptance Criteria
- 作为用户，我缩窄侧边栏后，窗口右侧的“当前”和数量仍应固定显示在最右侧。
- 作为用户，侧边栏再窄，优先被省略的也应该是窗口标题，而不是右侧信息。
- 作为用户，我不希望窗口行在不同宽度下切成单双行两种逻辑。
- 作为用户，右侧信息区未来如果加按钮，也应该继续固定在最右侧。

## Out of Scope
- 本版本不改标签列表的结构和交互。
- 本版本不改自动定位、窗口排序、标签高亮和竖条选中态。
- 本版本不引入新的按钮，仅为后续右侧按钮预留稳定布局规则。

## Risks / Dependencies
- 极端窄宽下，如果右侧信息项继续明显增加，标题可用空间会进一步缩小。
- 若未来右侧加入很长文本按钮，需要再次约束按钮文案长度，否则仍可能挤压标题区。

## Open Questions
- 当前无新的产品问题需要决策，后续若右侧加入按钮，再按“固定右侧动作区”规则扩展即可。
