# Chrome 侧边栏标签管理插件 PRD v2.0.3

## Executive Summary
- 本版本在 `v2.0.2` 的极简定位器基础上，修正“点击 Chrome 顶部标签栏后，侧边栏自动定位当前网页”的判定逻辑。
- 目标是让侧边栏始终滚动到**当前聚焦窗口**里的活动标签，而不是误滚到其他窗口的活动标签。

## Problem Statement
- 之前的实现会从整个侧边栏列表里查找第一个 `active` 标签作为滚动目标。
- 但 Chrome 是“每个窗口各自有一个活动标签”，所以在多窗口场景下，侧边栏可能滚到别的窗口的活动标签。
- 这会导致用户点击顶部标签栏后，侧边栏虽然发生了滚动，但滚到的并不是当前正在看的网页。

## Target Users
- 同时使用多个 Chrome 窗口，并依赖侧边栏快速定位当前网页的用户。
- 希望保留多窗口活动状态提示，但自动定位必须准确指向当前窗口网页的用户。

## Solution Overview
- 在选择器层新增 `selectCurrentActiveTabId(state)`，只从 `focusedWindowId` 对应窗口中寻找活动标签。
- 侧边栏滚动目标改为 `currentActiveTabId`，不再从整张列表里拿“第一个活动标签”。
- 视觉上继续保留每个窗口自己的活动标签状态：
  - 当前聚焦窗口中的活动标签使用最强高亮
  - 其他窗口中的活动标签使用较弱高亮
- 当前窗口自动展开逻辑继续沿用，确保被收起时也能先展开再定位。

## Success Metrics
- 点击 Chrome 顶部标签栏后，侧边栏能滚动到当前聚焦窗口中的对应标签。
- 多窗口场景下，其他窗口的活动标签不会再抢占滚动目标。
- 当前窗口活动标签的高亮强于其他窗口活动标签。
- 不影响现有窗口固定排序、点击跳转、图标显示和构建流程。

## User Stories / Acceptance Criteria
- 作为用户，我点击当前窗口顶部不同标签时，侧边栏会自动滑动到对应标签。
- 作为用户，我切换到另一个 Chrome 窗口中的顶部标签时，侧边栏会先展开该窗口，再滑动到那个网页。
- 作为用户，我仍然能看到其他窗口的活动标签，但它们不会影响当前网页的自动定位。
- 作为用户，当前窗口中的活动标签视觉上应该最明显。

## Out of Scope
- 本版本不改变后台消息协议。
- 本版本不改变窗口排序规则。
- 本版本不改变图标加载规则。

## Risks / Dependencies
- 该行为依赖后台继续正确同步 `focusedWindowId` 和各窗口标签 `active` 状态。
- 如果某些极端时刻 `focusedWindowId` 有值但对应窗口没有活动标签，侧边栏将不强行滚动，避免误定位。

## Open Questions
- 如果后续需要更强的“当前网页”视觉提示，可以再考虑在当前标签左侧增加极轻量状态点，但不应影响当前的极简布局。
