# Chrome 侧边栏标签管理插件 PRD v2.0.11

## Executive Summary
- 本版本继续修正窗口标题行在极窄宽度下的显示优先级。
- 目标是让网页标题区域不再保留多余最小宽度，随着侧边栏缩窄优先被省略为 `...`，把空间让给右侧“当前 / 数量”。

## Problem Statement
- `v2.0.10` 已将窗口行改为固定右侧槽位，但用户继续指出窄宽下核心诉求仍是“标题先缩，右侧信息保留”。
- 当前实现里，右侧信息区和标题区的收缩优先级仍不够明确，导致极窄时显示不符合预期。

## Target Users
- 会频繁缩窄 Chrome 侧边栏的用户。
- 更关心右侧状态信息稳定可见，而不是完整显示窗口标题的用户。

## Solution Overview
- 将窗口行第三列从固定 CSS 变量宽度改成 `max-content`，右侧信息区按内容自然占宽。
- 将窗口标题区进一步设为可收缩项，允许宽度继续缩小并稳定显示省略号。
- 去掉右侧信息区自己的隐藏裁切，让标题优先让位。

## Success Metrics
- 侧边栏缩窄时，窗口标题优先变短并显示 `...`。
- 右侧“当前 / 数量”比标题更稳定，不再轻易消失。
- 宽窄状态下窗口行结构仍保持稳定。

## User Stories / Acceptance Criteria
- 作为用户，我缩窄侧边栏时，窗口标题应该先被省略。
- 作为用户，我希望看到结尾 `...`，而不是右侧数量先没掉。
- 作为用户，右侧“当前 / 数量”应比标题更优先保留。

## Out of Scope
- 本版本不改标签列表结构和交互。
- 本版本不改自动定位、排序和高亮逻辑。

## Risks / Dependencies
- 如果未来右侧信息区继续增加更长内容，标题可用空间会进一步减少。
- 当侧边栏极端缩小时，标题可能很快只剩很短的可见片段，这是本版本主动选择的优先级结果。

## Open Questions
- 当前无新的产品决策问题；若后续右侧增加更多元素，再根据内容长度评估是否需要更严格的固定槽位宽度。
