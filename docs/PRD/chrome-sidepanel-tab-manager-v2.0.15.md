# Chrome 侧边栏标签管理插件 PRD v2.0.15

## Executive Summary
- 本版本微调网页行的左侧对齐。
- 目标是让网页图标和标题整体向左移动，与侧边栏左侧保持 `8px` 距离，同时保证当前选中竖条仍与内容对齐。

## Problem Statement
- 当前网页行左内边距过大，导致图标和标题整体离左边过远。
- 用户明确要求网页图标和标题向左移动，并与侧边栏左侧保持 `8px` 间距。

## Target Users
- 对侧边栏列表排版紧凑度和对齐感较敏感的用户。

## Solution Overview
- 将 `.tab-row` 左内边距从原值调整为 `8px`。
- 同步调整当前选中态竖条的 `left` 定位，使其与新的内容起点匹配。
- 不改窗口标题行布局，不改网页标题省略逻辑，不改交互逻辑。

## Success Metrics
- 网页图标和标题与左侧边缘的距离缩小到 `8px`。
- 当前选中态竖条仍与网页行左边缘对齐，不出现错位。
- 不影响 hover、高亮和自动定位行为。

## User Stories / Acceptance Criteria
- 作为用户，我希望网页图标和标题整体更靠左。
- 作为用户，我希望网页内容与侧边栏左边缘距离为 `8px`。
- 作为用户，我不希望当前选中竖条因为这次位移而错位。

## Out of Scope
- 本版本不改窗口标题行样式。
- 本版本不改自动定位、排序和高亮逻辑。

## Risks / Dependencies
- 左侧空间压缩后，需要保持竖条与内容之间仍有清晰层级；本版本通过同步调整竖条位置处理。
