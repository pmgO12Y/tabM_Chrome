# Chrome 侧边栏标签管理插件 PRD v1.0.17

## Executive Summary
- 本版本聚焦侧边栏启动稳定性，补齐启动失败可见性。
- 目标是避免出现“整块空白”或“无限转圈”但用户无法判断原因的情况。

## Problem Statement
- 近期多次改动后，侧边栏在某些浏览器状态下会出现启动页长时间停留、启动脚本失败后整页空白的问题。
- 当前问题不一定来自同一处业务逻辑，但共同风险是：失败没有被稳定暴露，用户无法区分是“加载慢”还是“脚本已失败”。

## Target Users
- 使用 Chrome Side Panel 的重度标签用户。
- 对扩展异常容忍度低、需要明确失败反馈的用户。

## Solution Overview
- 在 `sidepanel.html` 增加统一启动桥接层：
  - `ready()`：React 应用首屏挂载后主动移除启动遮罩。
  - `fatal(message)`：模块加载、未捕获异常、未处理 Promise 拒绝时，直接把启动遮罩切换为失败提示。
- 在 React 侧保留 Error Boundary，用于捕获渲染阶段错误，并通过启动桥接层显示失败信息。
- 启动超时提示保留，避免无限转圈。

## Success Metrics
- 启动失败时不再出现纯空白页。
- 启动失败时 4 秒内可见明确提示。
- React 渲染异常可见且可复现定位。

## User Stories / Acceptance Criteria
- 作为用户，当侧边栏启动正常时，我能直接看到应用内容，启动遮罩会被移除。
- 作为用户，当启动脚本报错时，我能看到“启动失败”的明确提示，而不是空白页。
- 作为用户，当异步初始化长时间无响应时，我能看到“加载时间过长”的提示，而不是无限转圈。

## Out of Scope
- 本版本不解决具体业务错误的根因，只先补启动可见性与兜底反馈。
- 本版本不调整标签数据模型与同步协议。

## Risks / Dependencies
- 如果错误发生在浏览器完全阻止页面脚本执行之前，仍可能无法由桥接层展示。
- 启动桥接层与 React 应用之间需要保持一致，避免重复控制启动遮罩。

## Open Questions
- 当前“彻底不显示”的具体根因，是否来自启动模块、标签读取，还是浏览器环境差异，仍需通过新增错误提示继续收敛。
